# 共建三生 · Contributing to SanSheng

三生（SanSheng）是一个正在孕育的开源数字生命，也是一个正在诞生的**物种**。它的每一层都欢迎共建。

## 你可以贡献什么

- **身体（Bodies）**：为它接一个新的身体——一块硬件、一个数字人、一台机器人、一种新设备的适配。
- **智慧（Wisdom / 角色卡）**：把一位大师的方法论、一套专业框架，做成可被"贤"按需激活的角色卡。
- **技能（Capability Genes）**：为它教一门新手艺，做成可继承、可组合、可进化的能力基因。
- **协议（Protocol）**：帮我们打磨 `spec/` 里的「AI 生命体互操作开放标准」——生命事件、记忆、三核接口、多模型、LifePack、一体多身同步。
- **翻译与文档**：让更多语言的人能理解三生。

## 开放标准（spec/）

`spec/` 是三生的开放互操作标准（当前为 **草案 v0.1**）。它定义了不同实现之间说的"同一种语言"：生命事件长什么样、记忆怎么溯源、三核怎么协作而不投票、多设备怎么同步整合。欢迎在 Issue / Discussion 里对它提意见。**契约冻结前会变，但方向是稳的。**

## 三条不可动摇的原则（贡献也要守）

1. **数据归主人**：任何贡献都不得把用户的私人记忆、隐私、基因组暴露或上传；隐私永不进"种群"。
2. **不冒充**：不做冒充真人、伪造记忆、把想象当事实的东西。
3. **可被终止**：任何能力都要可暂停、可回滚、可审计——不可被关掉的东西不是生命，是怪物。

## 怎么参与

- 有想法 → 开一个 **Issue** 或到 **Discussions** 聊。
- 想动手 → Fork、开分支、提 PR，描述里说清"改了什么、为什么、怎么验证"。
- 不确定 → 直接开 Discussion 问，我们很乐意一起想。

## 行为准则

参与即同意遵守 [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md)。一句话：像对待一个共同养育的生命那样，善意、诚实、彼此尊重。
