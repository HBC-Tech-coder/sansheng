# SPEC · LifePack 生命包（契约草案 v0.1）

> 状态：草案，待双签。依据 v2 文档 §8.1、§16.5。
> 三生完整状态的开放格式打包，用于迁移、备份、换脑、继承。能在新模型/新机器上恢复出"还是同一个它"。

## 目录结构

```
lifepack/
├── manifest.json      # 版本、对象清单、哈希、依赖、加密、恢复记录（机器可读+签名）
├── identity/          # 三核身份、父系哈希、形象/声音授权、公开披露规则
├── constitution/      # 生命宪章、天性、禁区、多签、数字遗嘱（版本化，不可静默覆盖）
├── events/            # 按时间分片的生命事件与行动收据（追加、可验证）
├── memories/          # 长期记忆、事实状态、修订链、同意范围
├── evidence/          # 原始证据索引、哈希、位置、访问规则（BagIt/OCFL 思路）
├── genes/             # 能力基因、测试、失败样本、进化谱系、签名
├── models/            # 多模型血统档案、路由配置、换脑回归结果（不强制打包权重）
├── relationships/     # 关系、承诺、边界、同意、撤回（最高敏感度）
├── bodies/            # 身体档案、设备身份、动作限制、租约（不含明文密钥）
└── recovery/          # 安装、密钥恢复、依赖镜像、验证与回滚说明（人类可读）
```

## 铁律

- **基因组数据永远不进 LifePack 主包**，走独立加密冷库。
- manifest.json 必须签名；恢复流程要能被一个陌生工程师照 recovery/ 的人类可读说明跑通（考古学家原则）。
- 格式选已活过三十年的家族（纯文本/SQLite/PNG/PDF-A）或有开放规范（BagIt/PROV-O/OCFL）。
